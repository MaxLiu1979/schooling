# 追求倍速增长的初建学校 · 选育用留体系

> Schooling 项目 · 人才管理模块
> 来源：科德高中融合部 × 《10倍人才》理论框架
> ClawHub Skill：`10x-talent-school`（`clawhub install 10x-talent-school`）

---

## 本模块解决的问题

本框架不是一本管理书的读书笔记。它是一所真实国际学校（科德高中融合部）在初建阶段，面对"如何在3年内从0到1搭建教师团队并实现倍速增长"这个命题时，使用的方法、踩过的坑、沉淀下来的操作标准。

### 核心命题

> 初建学校追求倍速增长，人才管理不是"选到完美的人"，
> 而是**用Schooling十部架构放大每个人的10倍产出**。

---

## 一、选人：识别"能陪学校跑过前三年的老师"

### 三个核心面试问题

**Q1：你怎么理解"初创学校"和"成熟学校"对老师要求的区别？**

✅ 通过信号：理解模糊感，对不确定性感兴趣，有从零搭建经历
❌ 危险信号：希望完整大纲和成熟考核体系

**Q2：如果你发现学校的某个流程不合理，你会怎么做？**

✅ 通过信号：主动沟通并提出改进方案
❌ 危险信号：按自己方式做，不按流程走

**Q3：你怎么理解AI在教学中的角色？**

✅ 通过信号：已经在用AI辅助工作，能区分辅助和替代
❌ 危险信号：完全排斥或完全依赖AI

### 选人优先级矩阵

见 SKILL.md 完整版本。

### 核心原则

初建学校宁可要"70分经验 + 90分共建意愿"的人，不要"90分经验 + 50分共建意愿"的人。

## 二、育人：在初建环境中快速成长

### 三阶段成长路径 × 十部Agent辅助

| 阶段 | 时间 | 目标 | 对应Agent |
|------|------|------|----------|
| 生存期 | 前2周 | 熟悉学生和流程 | 国子监Agent提供模板、通政司推送须知 |
| 建构期 | 3-8周 | 建立教学节奏 | 都察院推送预警、内阁推送驾驶舱 |
| 创造期 | 第9周起 | 输出特色 | 礼部协助家校沟通、技能沉淀为SKILL.md |

### 人才帽对话（入职第一个月）

```
不谈绩效，只谈：
1. 最有成就感的一件事 → 找高能量区
2. 哪个方面可以做比其他人好10倍 → 找差异化方向
3. 系统能帮你什么 → 让系统适配人才
```

## 三、用人：在十部架构中为老师找到最佳位置

### 十部 × 教师特质映射

| 部门 | 适合特质 | 初建学校关键作用 |
|------|---------|----------------|
| 国子监 | 逻辑强、结构化 | 搭建课程体系、制定SOP |
| 都察院 | 共情力强 | 家校信任、学生个案 |
| 礼部 | 表达力强 | 说明会、公众号、招生 |
| 通政司 | 执行力强 | 信息分发、进度追踪 |
| 内阁 | 系统思维 | 优先级、决策、向上管理 |

### 双帽工作法

每位老师 = 专业教学者 + 系统建设者

每周五下午不排课 → 写SOP、迭代流程 → 沉淀为 `framework/` 中的组织记忆

## 四、留人：倍速增长期的人才绑定

### 留人三角

愿景（真诚） + 成长感（每天有收获） + 心理安全感（犯错不挨罚）

### 实操清单

| 手段 | 做法 | 对应Agent |
|------|------|----------|
| 愿景对齐 | 每月一对一同步学校进展 | 内阁Agent |
| 成长可视化 | 让老师看到自己的SOP被使用 | 通政司Agent |
| 心理安全感 | 无惩罚反馈机制 | 都察院Agent |
| 利益相关 | 核心老师的成功=学校的成功 | 内阁Agent |
| 第三方效应 | schooling项目为老师提供背书 | GitHub/ClawHub数据 |
