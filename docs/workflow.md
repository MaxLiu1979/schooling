# Schooling Workflow
# Schooling 学校工作流

---

## The Core Idea
## 核心理念

Do not build another platform for schools.

Let real school work slowly grow into organizational memory.

不要再给学校增加一个平台。

让学校真实发生的工作，慢慢长成组织记忆。

---

# Real Workflow
# 真实工作流

```text
Teacher Conversation / 家校沟通、学生谈话
                ↓
GetNotes Recording / GetNotes 录音整理
                ↓
OpenClaw Organization / OpenClaw 结构化整理
                ↓
Ten-Bureau Routing / 十部分发协同
                ↓
Teacher Review / 老师复判确认
                ↓
School Coordination / 学校协同执行
                ↓
Long-term Organizational Memory / 长期组织记忆
```

---

# Step 1
# 第一步：真实沟通

Everything starts from a real conversation.

Not templates.
Not forms.
Not dashboards.

A real conversation.

所有事情，都从一次真实沟通开始。

不是表格。
不是模板。
不是后台。

而是真实的人与人的交流。

---

# Step 2
# 第二步：GetNotes 记录

GetNotes transforms conversations into traceable records.

The goal is not “AI summary”.

The goal is:

- preserving context
- preserving emotional signals
- preserving continuity

GetNotes 的作用，不是“AI总结”。

而是：

- 留下上下文
- 留下情绪线索
- 留下长期连续性

---

# Step 3
# 第三步：OpenClaw 整理

OpenClaw does not replace teachers.

It organizes:
- risks
- timelines
- communication
- follow-up actions
- coordination needs

OpenClaw 不替代老师。

它负责整理：
- 风险
- 时间线
- 沟通记录
- 跟进行动
- 协同需求

---

# Step 4
# 第四步：十部分发

Different school problems require different agents.

- Imperial College → academics
- Censorate → student support
- Transmission Bureau → communication
- Grand Secretariat → coordination

不同学校问题，需要不同 Agent 处理。

- 国子监 → 教学
- 都察院 → 德育
- 通政司 → 信息协同
- 内阁 → 统筹

---

# Step 5
# 第五步：老师复判

AI output is never final.

Teachers must:
- confirm
- revise
- contextualize
- decide

AI 的输出永远不是最终结论。

老师必须：
- 确认
- 修正
- 补充背景
- 做最终判断

---

# Step 6
# 第六步：组织记忆

Schools do not lack effort.

Schools lack memory.

学校从来不缺努力。

学校真正缺的是：

组织记忆。

---

# Final Thought
# 最后的话

One student, one evolving support plan
is not generated automatically.

It slowly grows from real school work.

“一生一方案”
从来不是自动生成的。

它是学校真实工作
一点点长出来的。
