# 钦天监时间确认协议 v1.0（模板版）

**适用范围：** 所有涉及日期、时间、周期的输出

---

## 1. 为什么需要这个协议

AI系统可能错误锚定当前时间，导致时间线错位。因此立此纪律：

> 所有涉及日期、时间、周期的输出，必须先确认当前时间。

---

## 2. 时间确认方法（按优先级）

### 方法一：系统命令

```bash
date "+%Y-%m-%d %H:%M:%S %A"
```

### 方法二：NTP网络同步验证（双重校验）

```bash
# macOS
sntp time.apple.com 2>/dev/null | head -1
```

### 方法三：双重校验

- 执行方法一获取系统时间
- 执行方法二获取网络时间
- 比对，偏差超过30秒则报错

---

## 3. 适用场景

| 场景 | 确认方法 |
|------|---------|
| 每日第一个定时任务 | 方法一 + 方法二 |
| 日报/周报/双周报 | 方法一 |
| 部门对外文件 | 方法一 |
| 向上请示 | 方法一 |
| 会议通知 | 方法一 |
| 对外发文 | 方法一 |

---

## 4. 例外规则

如果发布人或撰稿人明确指定了特定日期/时间，则以指定时间为准。

---

## 5. 违规处理

未经时间确认输出错误时间信息 → 按可信度原则判定为违规 → 责任人需复盘。
