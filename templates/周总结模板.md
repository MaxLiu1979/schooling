# 部门周总结 & 计划模板 v2.0

**核心设计：** 上期承诺回头看 → 本期交付 → 需决策上报 → 滚动追踪

---

```md
# [部门]｜上周总结 + 未来两周计划

提交：____年__月__日
周期：____月__日 → ____月__日

---

## 0｜一句话本期状态

（3行以内，说清本周核心交付+最大风险）

---

## 1｜上期承诺回头看

上期共承诺：____项
  · 已完成：____项
  · 未完成：____项（须逐条写原因）
  · 延期/后移：____项（须注新DDL）

### 1a｜未完成事项

| 未完成事项 | 原计划DDL | 未完成原因 | 新状态/新DDL |
|-----------|:--------:|-----------|:-----------:|
|           |          |           |             |

*原因类型参考：客观阻碍 / 缺资源 / 优先级调整 / 执行遗漏 / 依赖方未到
 禁止只写"推进中"——写清楚是"卡在哪"还是"还没做"*

### 1b｜上期判断修正（如有）

上期我们以为：______
实际情况是：______
认知更新：______

---

## 2｜本周重点工作回顾（≤5项）

| 事项 | Y对标 | 进展 | 状态 | 一句话判断 |
|------|-------|------|:----:|-----------|
|      |       |      |  G   |            |
|      |       |      |  A   |            |
|      |       |      |  R   |            |

**状态灯说明：**
- 🟢 G = 按预期推进，无需升级关注
- 🟡 A = 有滞后但有路径，需持续关注
- 🔴 R = 严重偏离预期，需决策介入

## 6｜下期回头看

下期写总结的人请先回答这3个问题：
1. 本期计划的那3件事，做完了吗？
2. 本期有什么判断需要修正？
3. 本期没提但后来发现严重的事？

```

---

## 填写纪律

1. 第1节必须先填——不回顾上期承诺，不写后面
2. "推进中"不得出现在第1节——要么已完成，要么延期+原因
3. 第3b节每项必须带"我建议"——上级不是来帮你想方案的
4. DDL就是DDL——到期没完成改新DDL时，旧DDL记录要保留
